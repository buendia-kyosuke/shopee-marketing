# 仕入れ・運用・発送 実務ガイド

## 基本スタンス：無在庫スタートから在庫ありへの段階移行

```
フェーズ1（無在庫）
  注文が入る → Amazonで購入 → 梱包 → 国際発送
  ↓
  月30〜50件安定して売れる商品が出たら
  ↓
フェーズ2（在庫あり切り替え）
  Amazonで数十個まとめ買い → 手元在庫 → 注文即日発送
```

メリット比較:

| 項目 | 無在庫 | 在庫あり |
|---|---|---|
| 初期投資 | ゼロ | 数万〜数十万円 |
| キャッシュフロー | 売上先行 → 仕入れ | 仕入れ先行 |
| 発送速度 | 3〜5日（Amazon到着待ち） | 翌日〜即日可能 |
| 在庫リスク | なし | 売れ残り損失あり |
| Shopee評価 | 発送遅延でマイナスリスク | 発送速度でプラス評価 |

## 価格差リサーチ手法

### 基本フィルター（ツールのスコアリングと連動）

```
Step1: Amazonで候補商品を探す
  → Amazonレビュー数: 100〜2,000件が狙い目
    （10件以下: 需要未証明、1万件超: 競合だらけ）

Step2: Shopeeで同商品の出品状況を確認
  → 競合出品数: 10件以下が理想
  → 競合の販売価格を確認（相場価格）

Step3: 価格差の確認
  → Amazon仕入れ値 × 2.5 < Shopee相場価格なら候補入り
  → 送料・手数料引いても利益が出るか試算
```

### 価格差が生まれやすい商品の特徴

```
1. 日本でしか流通していないブランド・メーカー
   → 対象国のAmazonや現地ECに存在しない

2. 使い方説明が必要な商品
   → 中国セラーが参入しにくい（説明が参入障壁になる）
   → 釣り具・理美容ツール・専門調理器具など

3. 日本語パッケージが価値になる商品
   → 「日本製」ブランドで現地セラーが太刀打ちできない

4. 中国からのコピー品が出にくいもの
   → 精密部品・ブランドものの小道具・伝統工芸系

5. 現地では需要があるが知られていない
   → Shopeeの商品コメントに「Is this available?」がある
   → Googleトレンドで対象国での検索量があるのに供給が薄い
```

### 価格差の試算例

```
利益 = Shopee販売価格 - Amazon仕入れ値 - 国内配送料（佐川） - SLS配送料 - Shopee手数料

※SLS配送料は現地通貨建て。購入者負担設定の場合はセラーの利益計算から除外できる。
※詳細送料表: research/market_comparison.md の「SLS送料一覧」参照

商品A（PH向け・300g商品）:
  Amazon仕入れ値: ¥800
  Shopee PH 競合相場: PHP600（¥1,800）
  国内配送料（佐川）: ¥200（目安）
  SLS配送料（PH・0.5kg以下）: PHP231
  Shopee手数料: 5.6% → ¥101
  ※SLS購入者負担: 利益 = ¥1,800 - ¥800 - ¥200 - ¥101 = ¥699 ✅
  ※SLSセラー負担: 利益 = ¥1,800 - ¥800 - ¥200 - ¥601（PHP231換算） - ¥101 = ¥98 ✗

商品B（SG向け・300g商品）:
  Amazon仕入れ値: ¥1,200
  Shopee SG 競合相場: SGD28（¥3,080）
  国内配送料（佐川）: ¥200（目安）
  SLS配送料（SG・0.5kg以下）: SGD3.15（約¥347）
  Shopee手数料: 14% → ¥431
  ※SLS購入者負担: 利益 = ¥3,080 - ¥1,200 - ¥200 - ¥431 = ¥1,249 ✅
  ※SLSセラー負担: 利益 = ¥3,080 - ¥1,200 - ¥200 - ¥347 - ¥431 = ¥902 ✅
```

## 発送方法：SLS（Shopee Logistics Service）

### SLSの仕組み

```
セラー自宅
  ↓
  佐川急便 東陽町 発送拠点（国内配送）
  ↓
  Shopeeルートで各国へ国際配送（SLS）
  ↓
  購入者
```

Shopeeが発行するSLSラベルを商品に貼って佐川で送るだけで、
国際配送まで一括でShopeeが処理する。

### 発生する送料2種類

| 費用 | 内容 | 目安 |
|---|---|---|
| 国内配送料 | 自宅 → 佐川 東陽町拠点 | 佐川の通常料金（重量・サイズ依存） |
| SLS配送料 | 東陽町拠点 → 購入者（国際） | 国・重量によって異なる（market_comparison.mdに国別一覧） |

> SLS配送料は購入者負担 or セラー負担（送料設定で選択）のどちらかに設定可能。

### 無在庫時の発送フロー

```
1. Shopeeで注文受付
2. ShopeeでSLSラベルを印刷
3. Amazonで商品を購入（自宅宛）
   ※「ギフト」設定で値段を非表示にする
   ※「Amazonロゴなし梱包」オプションを使用（可能な場合）
4. Amazon → 自宅 に届く（1〜2日）
5. 商品を梱包 → SLSラベルを貼付
6. 佐川急便で東陽町拠点へ発送
7. 追跡番号をShopeeに登録

目標発送日数: 注文から3日以内（Shopeeの評価基準）
```

### 在庫あり時の発送フロー（月30〜50件安定後）

```
1. Shopeeで注文受付
2. 手元在庫から商品ピック
3. 当日 or 翌日発送
4. 追跡番号をShopeeに登録

メリット: 発送速度がShopeeアルゴリズム評価に直結
```

## 在庫あり切り替えの判断基準

### 切り替えタイミング

```
以下を3ヶ月連続で満たした商品 → 在庫あり切り替えを検討:
  □ 月30件以上の安定注文
  □ 競合が増えていない（価格が維持できている）
  □ Amazonで安定して仕入れ可能（在庫切れがない）
  □ 商品あたり利益¥500以上
```

### 初回在庫数の決め方

```
月平均注文数 × 1.5ヶ月分 を初回ロットに

例: 月40件ペースなら 40 × 1.5 = 60個を発注
  → 在庫過多にならず、欠品リスクも低い水準
```

### 在庫ありに切り替えてはいけないケース

```
✗ 競合が急増中（価格競争が始まっている）
✗ Amazonの仕入れ価格が不安定
✗ 季節商品（オフシーズンに在庫を抱えるリスク）
✗ 発売から1年以内で需要が読めない
```

## 売上管理の実運用

### 管理すべき数値（商品単位）

```
商品ごとに記録:
  - Amazon仕入れ値（その時点の値段）
  - 発送コスト（実費）
  - Shopee販売価格・手数料
  - 実利益（手取り）
  - 月次注文数
  - 在庫状況（無在庫 / 在庫あり）
```

### ツールへの反映方針（buendia連携）

```
優先度1: 国別・商品別の実利益ダッシュボード
  → 「PHで月何件 × 利益いくら」を商品別に一覧
  → 無在庫→在庫あり切り替え判断に直接使う

優先度2: Amazon価格の変動アラート
  → 仕入れ価格が上がって利益が消えていないか監視
  → 無在庫の場合、仕入れ値変動が即利益に直撃する

優先度3: 発送コスト記録
  → 重量・サービス別に実費を記録
  → 商品ごとの適正発送方法を最適化するためのデータに
```

### 月次レビューの判断軸

```
毎月確認:
  □ 注文数TOP10商品の利益率確認
  □ 在庫あり切り替え候補の洗い出し（月30件超え商品）
  □ 利益率が落ちた商品の競合状況確認
  □ 出品停止候補（月3件以下・利益率10%以下）の整理
```
