# 自作ツール概要

## 基本仕様

| 項目 | 内容 |
|---|---|
| リポジトリ | github.com/buendia-kyosuke/pm（private） |
| フロントエンド | React |
| バックエンド | Python |
| Amazon連携 | スクレイピング |
| Shopee連携 | Shopee API |
| 対応国 | 切り替え可能（複数国対応済み） |
| 価格計算 | 利益率指定で自動計算 |

---

## 現在の機能フロー

```
Amazon商品スクレイピング
      ↓
価格・商品情報取得
      ↓
利益率に基づく販売価格計算
      ↓
Shopee API で出品（国切り替え可能）
```

---

## SG並走への対応状況

```
✅ 国切り替え可能 → SG即時対応可能
✅ 利益率自動計算 → PH/SG別マージン設定するだけ
✅ Shopee API → 安定した出品基盤
✅ React UI → 運用管理しやすい
```

**技術的障壁なし。SG向けマージン設定を追加すれば即並走可能。**

---

## 優先度別改善ロードマップ

### 優先度: 高

#### 1. 国別マージン設定の分離

```
現状: 単一のマージン設定（推測）
目標: 国別に独立したマージン設定

PH設定: 20〜30%
SG設定: 40〜60%
→ 同一商品でも国ごとに最適な価格で同時出品
```

#### 2. Shopee側競合価格モニタリング

```
現状: 出品して終わり
問題: 競合が安く出すと気づかずに機会損失・埋没

追加: 出品商品のShopee検索結果を定期取得
    → 自分の価格順位を監視
    → 値下げ競合が現れたらアラート
```

#### 3. 国別・商品別売上ダッシュボード

```
現状: PH/SGどちらが利益出てるか把握しにくい（推測）
追加: 国別・商品別の利益額ランキングをリアルタイム表示
    → 「どの商品をSGで伸ばすか」の意思決定が速くなる
    → 負け商品の早期撤退判断
```

### 優先度: 中

#### 4. 売れた商品の在庫切れアラート

```
売れる → Amazon在庫確認 → 切れそうなら通知 or 自動出品停止
```

#### 5. 価格自動追従

```
競合が値下げ → 自動で追随 or 下限価格で固定
（PHのせどり競争対策として有効）
```

#### 6. 同一商品のPH/SG同時出品

```
現状: 国を切り替えて別々に出品（推測）
目標: 1アクションで両国に同時出品
    → 運用工数を半減
```

### 優先度: 低（将来）

#### 7. Amazonスクレイピングのソース拡張

```
現在: Amazon.co.jp のみ（推測）
追加候補:
- Amazon.com（US）→ SG向け英語圏商品
- 楽天・Yahoo → Amazon未掲載の日本限定品
```

#### 8. ニッチ発掘スコアリング

```
自動スクリーニング条件:
- Amazonレビュー数: 100〜2,000件
- Shopee PH/SG 競合出品数: 10件以下
- Amazon価格 × 2.5 < Shopee相場価格
→ 「ニッチ候補リスト」を自動生成
```

## 戦略的な使い方の変化

```
現在:
Amazon商品を見つける → Shopee PHに出品

目指すべき姿:
Shopee PH/SG両方に同時出品
      ↓
売れた商品データを蓄積
      ↓
「PHで売れた商品をSGで高く売る」パターンの発見
      ↓
PHを実験場・SGを利益の場として機能分担
      ↓
勝ちパターンをMY/THに横展開
```
